- CI/CD 흐름의 위치와 목적 정리
  - 코드 저장소 <-> 이미지 저장소를 분리
  - 빌드와 실행을 분리하여 운영 안정성 확보

- 트리거(trigger)의 정확한 의미 재정의
  - (Before)
  - 실행을 요구하는 요청 
  
  - (After)
  - 자동화 시작의 조건이 만족되면 정의된 파이프라인을 실행

- YAML 파일의 역할
  - 자동화 규칙과 실행 계획을 선언하는 설정 문서

- ECR과 수평 확장의 연결
  - ECR : Docker 이미지 저장소
  - 코드 저장소와 빌드된 이미지 저장소를 분리
  - 

- 수직 확장 vs 수평 확장 정리
  - 수직 확장 : 서버 한 대 성능 증가
  - 성능이 증가된 서버의 한계 존재

  - 수평 확장 : 서버 수 증가
  - 장애 허용성을 높여 서버 간 의존도를 낮춤
  - 주의) 수평 확장의 의의는 성능이 아니라 안정성에 있음
