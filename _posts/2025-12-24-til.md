- CI/CD 흐름의 위치와 목적 정리
  - ECR : Docker 이미지 저장소
  - 여러 EC2가 동일한 이미지를 pull하여 실행함으로써 수평 확장의 기준점 역할
  - 빌드와 실행을 분리하여 배포 기준을 고정하고 운영 안정성 확보

- 트리거(trigger)의 정확한 의미 재정의
  - (Before)
  - 실행을 요구하는 요청 
  
  - (After)
  - 자동화 시작의 조건이 만족되면 (이 경우 GitHub Actions가) 정의된 파이프라인을 실행

- YAML 파일의 역할
  - 자동화 규칙과 실행 계획을 선언하는 설정 문서
  - 언제 실행할지(조건)와 무엇을 실행할지(계획)를 함께 정의

- 수직 확장 vs 수평 확장 정리
  - 수직 확장 : 서버 한 대 성능 증가, 성능이 증가된 서버의 한계 존재
  - 수평 확장 : 서버 수 증가
    - 장애 허용성을 높여 서버 간 의존도를 낮춤
    - 주의) 수평 확장의 의의는 성능이 아니라 안정성에 있음
