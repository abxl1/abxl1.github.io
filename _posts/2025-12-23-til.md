- 요청의 흐름 이해
  - (Before)
  - 요청을 앞단에서 처리하는 것은 미들웨어이다
  - 사용자의 요청이 곧바로 애플리케이션으로 들어간다

  - (After)
  - 요청은 인프라 파이프라인을 먼저 통과, 애플리케이션은 두 번째 검증
  - 미들웨어는 인프라 이후, 애플리케이션 내부 흐름의 일부
  - 애플리케이션 내부에서는 미들웨어가 비즈니스 로직보다 먼저 실행
  - 인프라 미들웨어(로드밸런서, 프록시 등)와
    애플리케이션 미들웨어(필터, 인터셉터 등)를 잘 구분할 것

- 동기 / 비동기의 결정 이유와 배치 작업
  - 동기 작업으로 처리 시 UX 저하 
  - 사용자가 기다릴 필요가 없는 작업을 비동기 / 배치로 분리


- 파비콘 적용
  - 책임 분리를 위해 assets 하위로 이동
  - minima 테마 한계로 파비콘 출력 불가
  - 루트로 이동하여 루트 경로를 다시 참조


- 콘솔 출력 형태 변경
  - mkdir 새 디렉터리 생성
  - touch 새 파일 생성
  - nano 파일 열고  수정
  - cat 파일 바로 보기
  - pwd 현재 위치 확인
  - cd 디렉터리 이동
  - ls 파일 / 디렉터리 목록 조회
  - source 설정 반영
